# Roadmap

## Phase 0 설계 확정

- 요구사항
- 위험 모델
- 포트폴리오 모델
- 보고서
- 아키텍처

완료 조건:
- 문서만으로 구현 시작 가능

## Phase 1 수동 보고서 검증

- 실제 데이터로 3회 작성
- 불필요한 항목 제거
- 판단 유용성 확인

상태:
- 완료

완료 조건:
- 5분 안에 핵심 판단 가능

## Phase 2 GitHub 하네스 구성

- 저장소 생성
- 문서 배치
- AGENTS 라우팅
- 설정 파일 초안

상태:
- 완료

완료 조건:
- 새 세션이 필요한 문서만 읽고 작업 가능

## Phase 3 데이터 자동 수집

- MVP 지표 6개
- 시장가격형: 기준일 현재값, 전주 대비 변화율, 4주 누적 변화율
- 정기발표형: 현재 발표치, 이전 발표치, 3개월 추세
- 공통 정규화 데이터 구조
- 수동 실행 화면 출력

완료 조건:
- 수동 실행으로 정규화된 데이터를 생성하고 화면에서 확인 가능

## Phase 4 위험 모델

- 임계값
- 영역 점수
- 전체 위험 단계
- 지속성

완료 조건:
- AI 없이 위험 판정

## Phase 5 포트폴리오 모델

- 테마 분류
- 민감도
- 약한 비중 보정
- 취약도

완료 조건:
- 상위 취약 테마 자동 출력

## Phase 6 AI 보고서

- 프롬프트
- 사실/해석 분리
- 특이사항만 설명
- 헷지 필요성 판정
- 필요한 경우에만 헷지 테마 1~2개
- Notion 연동 전에는 수동 실행 화면에만 보고서 출력

완료 조건:
- 자연어 보고서 자동 생성

## Phase 7 Notion 및 예약 실행

- Notion 저장
- GitHub Actions 예약
- 실행 오류 기록
- 중복 방지

완료 조건:
- 매주 자동 보고서

## Phase 8 중요 이벤트 경고

- 일간 또는 주기적 검사
- 경고 조건
- 중복 방지

## Phase 9 Telegram

- Bot
- 메시지 전송
- 주간/긴급 알림 분리
